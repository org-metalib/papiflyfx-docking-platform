# PR #11 Review Fixes - Execution Prompt

Addresses review comments from [org-metalib/papiflyfx-docking#11](https://github.com/org-metalib/papiflyfx-docking/pull/11).

---

```text
Fix the review comment on PR #11 for the papiflyfx-docking-archetype module.

Branch: archetype

Issue 1 (P1): AppLauncher calls non-existent App.main()

File: papiflyfx-docking-archetype/src/main/resources/archetype-resources/app/src/main/java/AppLauncher.java

Problem:
AppLauncher.main() calls App.main(args), but App extends javafx.application.Application
and does not define a main() method. Application itself does not provide one either.
This means every generated project will fail to compile.

Fix:
Change AppLauncher to use Application.launch(App.class, args), which is the standard
JavaFX trampoline pattern already used by the framework's own SampleLauncher
(see papiflyfx-docking-samples/src/main/java/org/metalib/papifly/fx/samples/SampleLauncher.java:23-24).

The corrected AppLauncher.java should be:

    package ${package};

    import javafx.application.Application;

    /**
     * Plain main() trampoline required because JavaFX Application classes
     * cannot be launched directly when the module path is not configured.
     */
    public class AppLauncher {
        public static void main(String[] args) {
            Application.launch(App.class, args);
        }
    }

After the fix:
1. Rebuild the archetype: ./mvnw -pl papiflyfx-docking-archetype -am clean install
2. Regenerate a test project in /tmp and verify it compiles:
   cd /tmp && rm -rf test-app
   mvn archetype:generate \
     -DarchetypeGroupId=org.metalib.papifly.docking \
     -DarchetypeArtifactId=papiflyfx-docking-archetype \
     -DarchetypeVersion=0.0.19-SNAPSHOT \
     -DgroupId=com.example \
     -DartifactId=test-app \
     -Dversion=0.1.0-SNAPSHOT \
     -Dpackage=com.example.testapp \
     -DinteractiveMode=false
   cd /tmp/test-app
   mvn wrapper:wrapper -Dmaven=3.9.12
   ./mvnw clean package
3. Commit with a message referencing the PR review.

Constraints:
- Only change AppLauncher.java in the archetype template. Do not modify App.java or other files.
- Use the same pattern as the framework's SampleLauncher.
```
