   cd /tmp
   mvn archetype:generate \
     -DarchetypeGroupId=org.metalib.papifly.docking \

     -DarchetypeVersion=0.0.19-SNAPSHOT \
     -DgroupId=com.example \

     -Dversion=0.1.0-SNAPSHOT \
     -Dpackage=com.example.testapp \

   cd /tmp/test-app
   mvn wrapper:wrapper -Dmaven=3.9.12

4. Build the generated project:
   ./mvnw clean package

   ./mvnw -Dtestfx.headless=true test

6. Verify the generated project structure matches the expected layout in summary.md.

7. If any step fails, diagnose and fix the root cause in the archetype templates.

   - Missing archetype-metadata.xml fileSet entries for new files
   - Package directory placement (__packageDir__ vs packaged="true")

1. Create src/test/resources/projects/basic/ in papiflyfx-docking-archetype.

4. Run: ./mvnw -pl papiflyfx-docking-archetype verify
